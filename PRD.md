## 🎯 产品概述

### 产品名称
**AI UI Design Comparator** - AI驱动的UI设计对比工具

### 产品定位
面向设计师和产品团队的AI图像生成对比工具，支持同时调用多个AI模型生成UI设计并进行并排对比。

### 目标用户
- **主要用户：** 产品设计师、UI/UX设计师
- **次要用户：** 前端开发者、产品经理
- **用户画像：** 需要快速迭代UI设计方案，追求效率和成本控制

### 核心问题
1. **效率低：** 逐个测试不同AI模型耗时长
2. **成本高：** 单独订阅多个AI工具成本高昂
3. **对比难：** 缺乏统一平台进行结果对比

### 解决方案
提供一个统一平台，一键调用多个AI模型，并排展示结果，实时显示成本，支持快速下载和重试。

---

## 💡 产品价值

### 用户价值
1. **提升效率 90%+** - 一次生成替代多次手动操作
2. **降低成本 99%** - 按需付费，避免多个订阅
3. **更好决策** - 并排对比帮助选择最佳方案

### 商业价值
1. **市场需求明确** - 设计师群体对AI工具需求旺盛
2. **差异化竞争** - 市场上缺少多模型对比工具
3. **扩展潜力** - 可扩展到其他AI能力对比（文本生成、代码生成等）

## 🎨 核心功能

### Phase 0（当前 - MVP）

#### 1. 多模型图像生成对比
**功能描述：**
- 支持同时调用4个以上AI模型
- 默认集成主流模型（Gemini、Flux、GPT-5、Claude）
- 并排展示生成结果
- 显示每个模型的耗时和成本

**验收标准：**
- ✅ 可同时选择多个模型
- ✅ 生成结果正确展示
- ✅ 显示每次调用的成本
- ✅ 失败情况有清晰提示

#### 2. 快速模板
**功能描述：**
- 预设常用UI设计场景（登录页、仪表盘、个人中心）
- 一键填充提示词
- 支持自定义修改

**验收标准：**
- ✅ 至少3个常用模板
- ✅ 点击模板自动填充提示词
- ✅ 用户可以修改模板内容

#### 3. 自定义模型管理
**功能描述：**
- 用户可添加OpenRouter支持的任意模型
- 配置模型ID、名称、预估成本
- 保存到本地存储

**验收标准：**
- ✅ 可添加、编辑、删除自定义模型
- ✅ 数据持久化存储
- ✅ 自定义模型可用于生成

#### 4. 结果管理
**功能描述：**
- 支持单张图片下载
- 失败结果保留并可重试
- 显示详细错误信息

**验收标准：**
- ✅ 图片可下载
- ✅ 失败卡片显示错误原因
- ✅ 可单独重试失败项

#### 5. API配置
**功能描述：**
- 用户输入OpenRouter API密钥
- 密钥存储在本地localStorage
- 配置界面友好

**验收标准：**
- ✅ 配置界面易用
- ✅ 密钥安全存储（仅本地）
- ✅ 配置错误有提示

---

### Phase 0.5（新增 - 模型中台）

#### 0. 模型中台 (Model Hub)
**功能描述：**
- 独立的管理后台页面（admin.html）
- 从OpenRouter同步最新模型列表
- 对模型进行4点验证测试（API调用/数据提取/响应时间/成本）
- 批量测试多个模型
- 只有验证通过的模型才能添加到首页

**验收标准：**
- [ ] 能从OpenRouter同步数百个模型
- [ ] 4个验证点全部实现并正确工作
- [ ] 批量测试功能正常
- [ ] 测试结果正确保存到Supabase
- [ ] 首页只显示已验证的模型

**4个验证标准：**
1. ✅ API调用成功（200状态码）
2. ✅ 成功提取到数据（图片/代码/文本）
3. ✅ 响应时间 < 60秒
4. ✅ 成本显示正确

**技术架构：**
- 前端：admin.html（纯HTML + Vanilla JS）
- 数据库：Supabase（models表 + test_logs表）
- 设计风格：Dark Editorial（与首页一致）

**实施计划：**
- Phase 1: 数据库设计（1小时）
- Phase 2: 基础页面（2小时）
- Phase 3: OpenRouter同步（2小时）
- Phase 4: 单模型测试（3小时）
- Phase 5: 批量测试（3小时）
- Phase 6: 首页集成（2小时）
- Phase 7: 优化完善（2小时）

**优先级：** 高（阻塞Phase 1其他功能）

## 🏗️ 技术架构

### Phase 0 架构

#### 前端
- **框架：** 纯HTML5 + Vanilla JavaScript
- **样式：** Tailwind CSS（CDN）
- **状态：** localStorage
- **部署：** 单HTML文件，直接运行

#### 后端
- **服务：** Node.js本地代理服务器
- **用途：** 仅用于开发环境CORS代理
- **生产：** 前端直接调用OpenRouter API

#### API集成
- **网关：** OpenRouter.ai
- **端点：** `/api/v1/chat/completions`
- **认证：** Bearer token

#### 数据存储
- **当前：** localStorage（API密钥、自定义模型、设置）
- **限制：** 无历史记录、无跨设备同步

### Phase 1 架构（计划）

#### 前端
- **框架：** React 18+
- **状态管理：** Zustand
- **样式：** Tailwind CSS
- **部署：** Vercel/Netlify

#### 后端
- **框架：** Node.js + Express
- **数据库：** PostgreSQL
- **认证：** JWT
- **存储：** Supabase Storage（图片）

#### 新增层
- **Model Adapter Layer：** 统一不同AI模型的接口差异
- **缓存层：** Redis（减少重复API调用）

## 🎨 设计规范

### 设计系统
**主题：** Dark Editorial

### 配色方案
```
背景色：
- Primary: #0c0a09 (深墨色)
- Raised: #1c1917 (浮起面)
- Overlay: #292524 (叠加层)

强调色：
- Accent: #f59e0b (琥珀金)
- Accent Hover: #fbbf24 (金黄)
- Success: #34d399 (薄荷绿)
- Error: #f87171 (珊瑚红)

文字色：
- Primary: #fafaf9 (主要文字)
- Secondary: #a8a29e (次要文字)
- Muted: #78716c (弱化文字)
```

### 字体
- **主字体：** Plus Jakarta Sans
- **权重：** 400（常规）、500（中等）、600（半粗）、700（粗体）

### 间距原则
- **宽松间距：** 组件间距至少16px
- **卡片内边距：** 24-32px
- **行高：** 1.6（正文）、1.2（标题）

### 交互设计
- **按钮状态：** Hover有明显视觉反馈（颜色变化 + 微阴影）
- **加载状态：** 显示进度条或骨架屏
- **错误提示：** 红色边框 + 图标 + 清晰文案

## ⚠️ 风险与挑战

### 技术风险
1. **API兼容性问题**
   - 风险：OpenRouter可能不完全兼容所有模型
   - 缓解：Phase 1实现Model Adapter Layer

2. **图片显示问题**
   - 风险：当前base64图片无法正确渲染
   - 缓解：测试不同模型返回格式，考虑服务端中转方案

3. **API费用超支**
   - 风险：用户大量调用导致成本失控
   - 缓解：添加使用限额和成本提醒

### 产品风险
1. **用户需求验证不足**
   - 风险：目标用户实际需求与假设不符
   - 缓解：快速发布MVP，收集真实用户反馈

2. **竞品快速跟进**
   - 风险：大厂推出类似功能
   - 缓解：建立用户社区，快速迭代差异化功能
